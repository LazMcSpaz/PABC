# v0.2+ Implementation Roadmap

Sequencing plan for the remaining v0.2+ design work. A map, not a recipe —
it gives the order, dependencies, files each phase touches, and the
cross-system gotchas. The **rules** live in the spec; read the cited
sections before planning a phase.

- **Design of record:** `docs/mechanical-spec-v0.1.md` (§16–§20).
- **Editor contract:** `docs/content-schema-v0.1.md`.

## Baseline: `main`

All work branches off **`main`** (the trunk) and returns via PR. `main`
already has §15, §16, and **§17 (the Tech Wheel) implemented**, plus an
in-progress radial-HUD UI. (An earlier scoping mistake targeted the stale
`claude/ashland-conquest-demo-BViiz` branch and assumed §17 was unbuilt —
that is corrected here.)

## Guardrails (every phase)

- Branch off `main`; PR back. No direct pushes to `main`.
- **Never edit `content/` or `src/game/content/`** — the content tool owns
  them. `src/game/content.js` and `config.js` are engine registries you may
  extend (schema fields, stub chips, constants); treat authored values as
  the content team's and coordinate.
- **Keep the demo playable after every phase.** No phase leaves the game
  unable to function (e.g. don't remove the Market without its replacement).
- **Verify before PR:** run the harness (`src/game/harness.js`) and drive
  the prototype in a browser; add harness coverage for your phase.
- TBD constants go in `config.js`, tunable, noted in the commit.
- The engine↔UI seam is **`src/prototype/engineAdapter.js`**. **UI now lives
  inside the radial HUD on `main`** — extend that system, don't build
  standalone widgets that duplicate it.

## Why this order (dependency logic)

- **§17 is done**, so the critical path now **starts at Loyalty**.
- **§18 is three phases, not one** — Loyalty (small, foundational),
  Influence/ZoC (medium), Diplomacy (huge capstone).
- **Loyalty (§18.2)** is foundational to Economy (§20 gating) and Fog (§19
  ZoC-vision) — so it goes first.
- **Economy (§20)** needs Loyalty (the gate) + Tech (done) and **retires the
  Market**.
- **Influence/ZoC (§18.3)** needs Loyalty; it's mostly a new module.
- **Fog (§19)** needs Influence (ZoC-vision) + Tech (Intelligence path, done).
- **Diplomacy (§18.4–18.13)** reads everything → last.

```
§17 ✓ ──────────────────────────────────┐ (Intelligence path, techLevel)
§18.2 Loyalty ─┬─► §20 Economy (gate; retires Market)   ┐
               └─► §18.3 Influence/ZoC ─► §19 Fog ───────┴─► §18.4-13 Diplomacy
```

**Order:** P1 Loyalty → (P2 Economy ∥ P3 Influence) → P4 Fog → P5 Diplomacy.

## P1 — Loyalty (§18.2)

**Goal:** replace foothold/decay with the 8-slice Loyalty pie. Small but
foundational — Economy and Fog both depend on it.

- **Depends on:** nothing (start now).
- **Spec:** §18.0 (canonical terms) + §18.2; supersedes §6.3.2.
- **Scope:** replace signed foothold `F` with `loyalty` (0–8, fixed
  ceiling). Control flips **only** on a lost contest or `loyalty == 0` (then
  peel one Control section/Upkeep to neutral). Loyalty rises while
  held/integrated, decays when neglected, floors at 0. Capital inert. Warn
  before any peel.
- **Touches:** `contest.js` (capture path → init Loyalty), `turn.js` (Upkeep
  Loyalty tick + peel, replacing the foothold tick), `setup.js` (location
  init), `config.js` (rates, danger threshold, peel cadence), `events.js`
  (`loyalty_changed`, `loyalty_failing`, `control_peeled`); UI: the Loyalty
  pie in the radial HUD's control meter; `engineAdapter.js` exposes
  `loyalty`.
- **Gotchas:** keep `loyalty` clean/queryable — Economy (gate) and Influence
  (feeds the field) read it. `town-hall`'s old "+foothold cap" is now a
  Loyalty-rate concept (content-owned; coordinate).
- **Done when:** harness shows a neglected capture peeling to neutral via
  Loyalty 0, a garrisoned one holding, and the warning firing first.

## P2 — Economy & City Development / retire the Market (§20)

**Goal:** chips become the output of the economy; the Market is removed.
Most disruptive phase — keep the demo playable (the build-system replaces
the Market in the *same* change; don't ship a market-less, build-less
state).

- **Depends on:** Tech (done) + Loyalty (P1) for the gate. The build-system,
  Output slider, and market-removal are **Loyalty-independent**; only the
  Loyalty-gating rung (§20.6) and the Loyalty-drop chip rule (§20.8) need
  P1 — defer those and wire them at integration.
- **Spec:** §20 (all); §20.6 (gating display *contract*) and §20.11 (engine
  map) are exact.
- **Scope:** remove Market + Acquire + churn + the Market HUD; add per-
  Location Output + the guns/butter build slider; build chips into slots and
  upgrade in place; unit-chips need a stationed unit; rush-building;
  selective per-chip `upkeep` (dormant if unpaid); Tech×Loyalty gating with
  the §20.6 display rule.
- **Touches:** **remove** `market.js`/`churnMarket` (`turn.js`), Acquire
  (`actions.js`), market seeding (`setup.js`), `marketRow`/`marketDeck`
  (`effects.js`), Market HUD; **add** Output/slider/`activeBuild` to Location
  state, the §20.11 effects/events, chip schema fields in `content.js`
  (`buildCost`/`upgradesTo`/`loyaltyReq`/`upkeep` + stub upgrade chains).
  Rework the `turn.js` Upkeep so the slider **replaces `collectProduction`**.
  Big UI: slot-click build menu + chip-click upgrade view in the radial HUD.
- **Gotchas:** `collectProduction` is gone — route all scrap income through
  Output. Reuse the existing `{techLevel ≥ 1/3/5}` gate. The AI **must** set
  sliders and pick builds.
- **Done when:** a full game runs with no Market — build/upgrade off Output,
  rush works, an unpaid-upkeep chip goes dormant, the §20.6 display rule
  behaves in-browser.

## P3 — Influence & Zone of Control (§18.3)

**Goal:** the soft territorial field. Light-touch — **not** wired into
contest math or yield. Mostly a new module → low file contention.

- **Depends on:** Loyalty (P1) — Loyalty feeds local influence.
- **Spec:** §18.3 (+ §18.0); heed the **ZoC ≠ Vision** warning (separate
  sets; Fog builds Vision later).
- **Scope:** deterministic per-faction Influence scalar field + a derived ZoC
  owner map in `state.world`, recomputed (no dice) on control/Loyalty/chip
  change. Uses: reinforcement routing (connect to §16.5 pathing), encounter-
  reveal `condition` hook; diplomatic reading deferred to P5.
- **Touches:** new `src/game/influence.js`; `config.js` (range, falloff,
  threshold); `events.js` (`zone_changed`); UI ZoC overlay; adapter exposes
  the ZoC map. Influence chips are content.
- **Done when:** capturing/integrating shifts ZoC borders and reinforcement
  routing respects them.

## P4 — Exploration & Fog of War (§19)

**Goal:** the fourth X. Compute-heavy; the AI plays under the same fog
(biggest cost is the AI refactor).

- **Depends on:** Influence (P3, ZoC-vision), Tech (done, Intelligence path).
  Ambush touches the shipped §16 combat — don't regress it.
- **Spec:** §19 (all); §19.11 engine map, §19.2 persistence rule.
- **Scope:** per-faction `state.visibility[fid]` (explored/visible/memory +
  ghosts); Vision stat + Location/ZoC contribution; Detection; LoS over
  elevation/cover; concealment + hidden armies; ambush (adds a §16.6 term,
  suppresses the §10 reaction window); hidden encounter hexes; scout/
  watchtower chips (content); the Intelligence-path branch effects.
- **Touches:** new `visibility.js` (incremental recompute = scale guard);
  `board.js` (elevation/cover); `contest.js` (ambush); `actions.js` (reveal
  on move); `events.js`; new effects; big fog UI + ghosts; **`ai.js` refactor
  to plan on its own visibility/memory.**
- **Done when:** fog renders per faction, ghosts persist, an ambush resolves
  with the bonus + no defender reaction, and an AI scouts and gets surprised.

## P5 — Diplomacy capstone (§18.4–18.13)

**Goal:** factions, reputation, deals, coalitions, vassalage, the
Recognition victory. Largest AI lift; reads every prior phase.

- **Depends on:** Loyalty, Influence (reception), Fog (intel-deals/shared
  vision), Economy (scrap-deals), Tech.
- **Spec:** §18.4–§18.13.
- **Scope:** non-cosmetic faction model + faction↔faction Standing (extend
  `state.factionStanding`); player Menace + Honor; derived Tolerance/trust-
  floor; the deal-basket primitive; diplomatic actions; AI valuation +
  AI-to-AI + coalitions (threat = Menace **and** power/VP lead); vassalage;
  the Recognition win.
- **Touches:** extend `standing.js`; new `diplomacy.js`; win-condition
  evaluator; reuse §15 encounter delivery for proposals/pact-calls; reuse
  `ADJUST_STANDING`; many effects/events; a Diplomacy screen in the HUD;
  **the big `ai.js` lift.**
- **Done when:** AI-vs-AI war forms and resolves without the human;
  Menace/Honor move correctly; a coalition forms against a runaway; a vassal
  can be taken and can rebel; Recognition can be won and is gated by
  Menace/Honor.

## Cross-cutting (every phase)

- **AI (`ai.js`)** grows each phase; P2, P4, P5 are the big lifts. Never
  leave the AI unable to play the new system.
- **Adapter (`engineAdapter.js`)** is the only place React reads engine
  state. P4 makes it per-viewing-player (fog).
- **UI lives in the radial HUD** now — integrate; don't rebuild.
- **Events:** each phase adds names to `events.js`; surface the meaningful
  ones in the HUD's event feed.
- **Testing:** extend `harness.js` per phase; confirm features in-browser
  (type-check proves correctness, not feature).
- **Content boundary:** `content/` and `src/game/content/` are off-limits.

## Parallel windows

See `docs/parallel-agent-briefs.md`. In short: P1 (Loyalty) is the only
cleanly-startable phase right now; P2 (Economy) can run alongside it with a
coordination protocol (shared `turn.js` Upkeep), and design-table authoring
runs cold. After P1+P2 land, **P3 (Influence) is near-conflict-free** (new
module) and pairs well with P4 prep. P5 is the solo capstone.
